계층(색상)

계층 구조

  • primary: 섹션의 최종 목표를 달성하는 단 하나의 메인 제출 버튼 (등록, 저장, 제출, 완료)
  • secondary: Primary를 보조하거나 자주 사용되는 중요 기능으로, 아웃라인 스타일
  • tertiary: 가끔 사용하는 부가 기능으로, 시각적 강조 최소화 (관리, 추가, 수정, 초기화, 중간저장, 다운로드)
  • inverse: 데이터 조회/검색의 메인 버튼으로, 검색 폼 하단이나 필터 섹션에서만 사용 (조회하기, 검색, 필터 적용)
  • neutral: 긍정도 부정도 아닌 중립 액션으로, 취소나 일반 편집에 사용 (취소, 닫기, 수정)
  • subtle: 시각적으로 거의 보이지 않는 최소 강조 버튼으로, 외부 링크나 비중요 액션에 사용

CSS 셀렉터

마크업 가이드, CSS 선택자에 대한 표는 필수, 선택의 구성으로 이루어져 있으며, 세부 항목으로는 전체영역, 색상, primary, secondary, tertiary, 사이즈, 더 작게, 작게, 중간, 크게, 더 크게, 텍스트형, 아이콘만 있는 버튼, 기본, border 및 bg 항목을 포함한다.
필수 선택
전체영역 .btn_base 계층(색상) primary: 가장 중요한 액션 (저장, 확인, 제출 등) .primary
secondary: 보조 액션, 아웃라인 스타일 (취소, 닫기 등) .secondary
tertiary: 세 번째 우선순위, 일반적인 액션 .tertiary
inverse: 반전 스타일, 어두운 배경용 .inverse
neutral: 중립적인 액션, 보통 강조 .neutral
subtle: 시각적 강조도가 낮은 액션 (더보기, 옵션 등) .subtle
